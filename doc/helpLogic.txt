Краткое описание некоторых процессов.
1. Ссылки Link & invite
Данные ссылки игнорируют часть проверок в preheader.php и попадая на страницу invite.php распознаются по типу ссылка/приглашение, далее получает ИД бланка по пермалинку из БД, после чего открывает бланк пригдашения (не требуя авторизации).
2. Авторизация
При авторизации после сравнении логина и пароля добавляется номер сессии и ид админа в таблицу сессий, должна удаляться любая запись не активная более года (в таблице есть поле записывающее дата последнего посещения).
При открытии страницы запрашивается ИД сессии сравниваетя с записями в БД и достаётся нужный пользователь.
При выходе из аккаунта сессия удаляется.
При выходе "ото всюду" удаляются все сессии пользователя.

2. Reference system (HELP)
Ссылки на гугл документы в системе помощи создаются на странице reference. Добавление ссылок возможны для страниц которые добавлены в таблицу PAGE. Можно подгружать в список страниц в комбобоксе страниц на странице reference страницы из таблицы  custom_page.

3. Permalinks вида invites?link= и invites?invited=

4. Архивирование мероприятий работает для добавления в архивный список мероприятий через форму и как процесс своего рода архивирования Активных Синхронизированных прошедших событий на странице события
