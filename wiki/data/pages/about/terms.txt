====== Термины и определения ======

На этой странице собраны основные термины, которые предпочтительно использовать в работе для взаимопонимания между участниками команд регистрации и разработчиками системы регистрации.

----

===== Компоненты системы регистрации =====

**Система регистрации** -- информационная система для регистрации на различные мероприятия ((Обучения, конференции, семинары, поездки и т.п.)) в русскоязычном мире. Система включает в себя //программу регистрации// ((Работает на платформе 1С:Предприятие)) и //сайт регистрации//.((Размещён по адресу http://reg-page.ru)) Архитектура системы описана [[about:technology|здесь]].

**Программа регистрации** -- это прикладное решение для автоматизации учёта мероприятий ((Обучения, конференции, семинары, поездки и т.п.)) их участников, а также связанной с ними информации. С //базой регистрации// работают команды регистрации.

**Сайт регистрации** -- веб-ресурс,((Размещён по адресу http://reg-page.ru)) предназначенный для онлайн-регистрации на различные мероприятия.((Обучения, конференции, семинары, поездки и т.п.)) С //сайтом регистрации// работают пользователи ((посетители сайта, получившие приглашение и пароль для самостоятельной регистрации.)) и администраторы сайта.((Ответственные в различных городах, получившие административные права.)) Данные об участниках, зарегистрированных через //сайт регистрации//, обрабатываются в //программе регистрации//.

^ {{:about:attention.png |}}В этом руководстве собрана справочная информация о работе //программы регистрации//. Справочная информация о работе //сайта регистрации// находится [[http://help.reg-page.ru|здесь]]. ^

===== Объекты программы регистрации =====

//Программа регистрации// включает в себя различные объекты, назначение и особенности которых важно понимать для эффективной работы в базе.

**Справочник** представляет собой таблицу для хранения в базе условно-постоянных данных.((Это данные, которые обычно не изменяются или изменяются довольно редко)) Например, справочник "Мероприятия" предназначен для хранения данных о мероприятиях (Название, дата начала, дата окончания, сумма взноса и т.д.), справочник "Участники" предназначен для хранения данных об участниках мероприятий (Ф.И.О., дата рождения, категория и т.д.).

Каждый //справочник// включает в себя //элементы// справочника.((Элемент справочника -- это, по сути, одна строка таблицы, которой является справочник.)) //Ссылки// на элементы справочников используются в различных объектах базы регистрации. Благодаря этому, во-первых, обеспечивается быстрый выбор значений при заполнении полей на экранных формах и, во-вторых, обеспечивается однозначность данных.

**Перечисление** -- это краткий список, который в отличие от справочника не предполагает редактирования. Его назначение похоже на назначение справочника.((По сути, справочник - это таблица из нескольких колонок, а перечисление - это таблица из одной колонки, т.е. простой список.)) Примеры перечислений: "Пол участника" (мужской, женский), "Виды регистрации" (предварительная, фактическая).

Учет в программе регистрации ведётся в разрезе справочников и перечислений.

**Документ** представляет собой бланк, который предназначен для отражения в базе информации о различных событиях: регистрация участника на конкретное мероприятие, оформление регистрационного взноса, списание денежных средств и т.п. //Ссылки// на документы используются в различных объектах программы регистрации.

**Регистр** представляет собой таблицу для: 
  * хранения данных, по которым ведётся учет прихода и расхода (регистры накопления);
  * учёта данных, которые изменяются в течение времени (периодические регистры);
  * записи каких-либо временных данных.

Записи в регистр делаются с помощью документов или непосредственно.

**Обработка** -- это объект программы, который предназначен не для хранения, а для обработки различных данных. Например, с помощью обработок осуществляются поиск участников в базе данных, одновременное создание нескольких документов (групповая регистрация) и другие действия.

**Отчёт** -- это объект программы, предназначенный для вывода сводной информации в виде таблиц, списков и диаграмм. Отчёт, как и обработка, не предназначен для хранения каких-либо данных.

===== Термины, используемые в ходе регистрации =====

**Участник** - человек, который регистрируется на мероприятие.

**Персональные данные** - условно-постоянная информация об участнике: дата рождения, пол, населённый пункт, категория и т.п.

**Категория участника** - один из параметров группирования участников: школьники, студенты, святые в церковной жизни, верующие и др.

**Регистрационные данные** - данные об участнике в связи с определённым мероприятием: дата приезда, дата отъезда, статус участия и т.д.

**Статус участия** - один из параметров группирования зарегистрированных участников: участник, служащий, говорящий брат и др.

**Карточка участника** - элемент справочника "Участники" с персональными данными.

**Бланк регистрации** - документ, включающий в себя персональные данные (из карточки участника) и регистрационные данные (на текущее мероприятие).
