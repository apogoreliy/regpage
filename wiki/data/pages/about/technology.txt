====== Архитектура системы ======

===== Основные компоненты =====

//Система регистрации// включает в себя //базу регистрации// и //сайт регистрации//.((Размещён по адресу http://reg-page.ru))

**База регистрации** является прикладным решением, созданным на платформе 1С:Предприятие 8.2 (управляемое приложение) и работающим по [[saas|технологии SaaS]] на [[http://www.rent1c.ru/about/datacenter|сервере провайдера]]. Доступ к базе регистрации, расположенной на сервере, осуществляется через [[about:thinclient|тонкий клиент]], установленный на локальном компьютере пользователя.

**Сайт регистрации**, написанный на PHP, размещён на хостинге одного из провайдеров. На сайт регистрации из базы регистрации выгружаются данные, необходимые для процедуры регистрации: активные мероприятия, список участников и пр. С сайтом регистрации работают пользователи ((Посетители сайта, получившие приглашение и пароль для доступа к бланку регистрации.)) и администраторы сайта.((Ответственные в различных городах, получившие административные права.)) Все данные сайта регистрации считываются и обрабатываются базой регистрации через Интернет. У сайта нет инструментов доступа к базе регистрации.

===== Общие принципы =====

База регистрации является общей для различных команд регистрации, которые могут работать с данными базы одновременно. У каждой команды и у каждого пользователя базы имеются свои права доступа и, соответственно, свой интерфейс.

Количество одновременно работающих в базе пользователей((Т.е. членов команды регистрации.)) определяется количеством каналов доступа, которые выделены команде регистрации. Например, если команде регистрации выделено два канала, это значит, что одновременно могут работать с базой два человека. При этом попеременно через один канал с базой может работать неограниченное количество человек.

[[objects:catalogs:event|Мероприятия]], [[objects:documents:regform|бланки регистрации]] и другие объекты той или иной команды регистрации доступны другим командам для просмотра, но не для редактирования. Однако [[objects:catalogs:member|картотека участников]] является общей для всех команд регистрации. Поэтому те святые, у которых есть право создания и редактирования карточек участников, должны быть особенно внимательны в работе с картотекой. В частности, перед тем, как добавить в базу нового участника, необходимо убедиться в том, что этот участник не был создан ранее или не поменял фамилию. Сделать это можно с помощью [[objects:dataprocessors:membersearch|окно поиска участников]].

===== Безопасность данных =====

База данных физически хранится на сервере в промышленном дата-центре, который находится на территории закрытого режимного объекта со строго ограниченным доступом. Вся информация при работе через Интернет передается в зашифрованном виде по защищенному каналу. Аналогичную схему шифрования с открытыми и закрытыми ключами используют банки в системах "банк-клиент".

Ежедневно в автоматическом режиме выполняется процедура резервного копирования информационной базы. Техническим сопровождением системы занимаются системные администраторы компании-провайдера.
